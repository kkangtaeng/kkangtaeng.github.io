## Sequelize

### Sequelize-cli 설치

```jsx
npm install --save-dev sequelize-cli
```

위 명령어를 입력해 설치한다.

### Sequelize-cli 초기세팅

```jsx
npx sequelize-cli init
```

위 명령어를 입력하면 config, models, migrations, seeders 폴더가 생성된다.

config에는 처음에 config.json파일로

```jsx
{
  "development": {
    "username": "root",
    "password": null,
    "database": "database_development",
    "host": "127.0.0.1",
    "dialect": "mysql"
  },
  "test": {
    "username": "root",
    "password": null,
    "database": "database_test",
    "host": "127.0.0.1",
    "dialect": "mysql"
  },
  "production": {
    "username": "root",
    "password": null,
    "database": "database_production",
    "host": "127.0.0.1",
    "dialect": "mysql"
  }
}
```

위와같이 생성된다. 하지만 환경변수를 사용하게 위해 `.env`파일을 생성 후

```jsx
DATABASE_USERNAME = root;
DATABASE_PASSWORD = qwer1234;
DATABASE_NAME = lohas;
```

위와 같이 만들고 config.json 파일을 config.js로 변경 후

```jsx
const dotenv = require("dotenv");
dotenv.config();

module.exports = {
  development: {
    username: process.env.DATABASE_USERNAME,
    password: process.env.DATABASE_PASSWORD,
    database: process.env.DATABASE_NAME,
    host: "127.0.0.1",
    dialect: "mysql",
  },
  test: {
    username: process.env.DATABASE_USERNAME,
    password: process.env.DATABASE_PASSWORD,
    database: process.env.DATABASE_NAME,
    host: "127.0.0.1",
    dialect: "mysql",
  },
  production: {
    username: process.env.DATABASE_USERNAME,
    password: process.env.DATABASE_PASSWORD,
    database: process.env.DATABASE_NAME,
    host: "127.0.0.1",
    dialect: "mysql",
  },
};
```

위와 같이 작성해 주었다. models/index.js 파일에

```jsx
const config = require(__dirname + "/../config/config.js")[env];
```

이것도 js로 변경해야한다!!

### models 생성 및 migration

```jsx
// 예시
npx sequelize-cli model:generate --name 이름 --attributes 속성:값

npx sequelize-cli model:generate --name portfolio --attributes title:string,message:text,size:integer...
```

위의 예시와 같이 입력하면 생성할 수 있다. 신기했던건 varchar가 아닌 string을 입력하는 거였다.  
찾아보니 sequelize의 타입 정의와 MYSQL의 타입 정의가 달랐다 그 이유는 여러 개의 관계형 데이터베이스 시스템을 호환하기 위해서 공통의 문법으로 통일 했기 때문이라고 한다!!

다시 돌아와서 위처럼 model을 생성하게 되면 migration도 같이 만들어지게 된다.  
추가적인 기능이 필요하다면 models와 migrations 파일들을 수정해야한다.(defaultValue 등)

```jsx
npx sequelize-cli db:migrate
```

이제 만들어놓은 데이터베이스에 migration 하기 위해서 위 명령어를 입력하면 모델이 데이터베이스에 생성된다.

### seed 생성 및 연결

```jsx
//예시
npx sequelize-cli seed:generate --name 이름

npx sequelize-cli seed:generate --name portfolio
```

위와 같이 시드를 생성하게 되면 `seeders`폴더에 파일이 생성되고

```jsx
module.exports = {
  async up(queryInterface, Sequelize) {
    await queryInterface.bulkInsert("portfolios", [
      {
        title: "인천 아파트",
        type: "아파트",
        size: 43,
        location: "인천",
        before_image: "",
        after_image: "",
        message: "이번 아파트 시공 사진입니다. 너무 잘됬죠 ㅎㅎ",
        createdAt: new Date(),
        updatedAt: new Date(),
      },
    ]);
  },
```

위와같이 수정 후

```jsx
npx sequelize-cli db:seed:all
```

위 명령어를 입력하면 데이터베이스에 seed가 추가된다.
