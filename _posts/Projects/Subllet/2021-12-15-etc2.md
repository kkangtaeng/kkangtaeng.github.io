## - AWS S3 이미지 서버 구축

### 1. IAM 역활 설정

IAM 메뉴에 들어와 AWS에 접근할 수 있는 권한을 먼저 부여해야한다.

![스크린샷_2021-12-14_23 59 22](https://user-images.githubusercontent.com/85768164/146034411-df41f11c-c66c-4b82-9f5d-1bddd3cf9190.png)

먼저 IAM 페이지로 들어와 `역할`탭을 클릭 후, `역할 만들기`를 클릭한다.

![스크린샷_2021-12-15_00 03 09](https://user-images.githubusercontent.com/85768164/146034422-5d63d87b-2af6-4798-9981-2d9c74660bb4.png)

그 다음, `S3`를 선택하고 사용사례 선택도 `S3`를 선택한다.

![스크린샷_2021-12-15_00 05 16](https://user-images.githubusercontent.com/85768164/146034429-197e0753-d276-4ce7-b661-3a21d04d92aa.png)

그 다음, `AmazonS3FullAccess`를 체크 후 다음으로 넘어가서 태그는 건너뛴다.

![스크린샷_2021-12-15_00 07 01](https://user-images.githubusercontent.com/85768164/146034432-eae99c50-3837-4cd1-a5d6-0270aa489232.png)

그리고 역할 이름을 작성한다. (역할 이름은 존재하지 않는걸루!)

### 2. Cognito 권한 부여

먼저 Cognito 페이지로 들어가 `자격 증명 풀관리`를 클릭해서 들어간다.

그다음, `새 자격 증명 풀 만들기`를 클릭한다.

![스크린샷_2021-12-15_00 10 58](https://user-images.githubusercontent.com/85768164/146034434-2af05117-4d09-47df-b2c9-fb6e56a632e5.png)

그 다음, `자격증명 풀 이름`을 작성하고, `인증되지 않은 자격 증명에 대한 액세스 활성화` 를 체크하고 풀 생성을 클릭한다.

![스크린샷_2021-12-15_00 16 09](https://user-images.githubusercontent.com/85768164/146034438-b8dd7769-2501-4ef8-b75c-cd1cc5b223c1.png)

그 다음, `세부정보 보기`를 클릭, `정책 문서 보기` 클릭, `편집`을 눌러

```jsx
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["cognito-sync:*", "cognito-identity:*", "s3:*"],
      "Resource": ["*"]
    }
  ]
}
```

위 코드로 변경한다.

![스크린샷_2021-12-15_00 17 56](https://user-images.githubusercontent.com/85768164/146034442-f9de9509-9481-423c-8121-e25a7e249ad2.png)

위처럼 자격 증명 풀ID와 리전을 받을 수 있다.

### S3 버킷 생성

S3 페이지로 들어가 `버킷 만들기`를 클릭하고 버킷 이름을 작성한다. (버킷 이름은 고유한걸로!!)

![스크린샷_2021-12-15_00 21 37](https://user-images.githubusercontent.com/85768164/146034444-a04f1aa4-3fe7-4e8e-b901-48a8104acfbd.png)

그리고 `모든 퍼블릭 엑세스 차단`을 체크 해체하고 `버킷 만들기`를 클릭한다.

![스크린샷_2021-12-15_00 25 05](https://user-images.githubusercontent.com/85768164/146034445-651eeca8-3f1b-4847-99dc-17df02c4d26c.png)

그 다음, 버킷으로 들어가 `권한`탭을 클릭 후, 버킷 정책에서 `편집`을 클릭하고

```jsx
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AddPerm",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": "arn:aws:s3:::<버킷 이름>/*"
    }
  ]
}

<버킷 이름>은 아까 만든 버킷 이름!!!
```

위 코드로 변경한다.

![스크린샷_2021-12-15_00 27 17](https://user-images.githubusercontent.com/85768164/146034448-a4463ac1-43ab-4b46-b922-d305c483ac8b.png)

그리고, 밑에 스크롤을 내려 CORS도 `편집`을 클릭하고

```jsx
[
  {
    AllowedOrigins: ["*"],
    AllowedMethods: ["GET", "PUT", "POST", "HEAD"],
    AllowedHeaders: ["*"],
    ExposeHeaders: [
      "x-amz-server-side-encryption",
      "x-amz-request-id",
      "x-amz-id-2",
    ],
    MaxAgeSeconds: 3000,
  },
];
```

위 코드로 변경한다.

## - AWS S3 이미지 서버 연결

### 1. AWS SDK

먼저 aws-sdk 모듈을 설치한다.

```jsx
const AWS = require("aws-sdk");

AWS.config.update({
  region: "ap-northeast-2", // 버킷의 리전
  credentials: new AWS.CognitoIdentityCredentials({
    IdentityPoolId: "ap-northeast-2:553da489-28ff-4eb6-b0ba-1187a7a08c29", // 자격 증명 풀ID
  }),
});
```

위 코드에서 region 과 IdentityPoolId 를 넣어준다.

### 2. <input type="file">

클라이언트에서 파일 업로드를 하기 위해서 위 태그를 사용한다.

```jsx
const [profile, setProfile] = useState(UserInfo.profile);

//...

const AWS = require("aws-sdk");

AWS.config.update({
  region: "ap-northeast-2", // 버킷의 리전
  credentials: new AWS.CognitoIdentityCredentials({
    IdentityPoolId: "ap-northeast-2:553da489-28ff-4eb6-b0ba-1187a7a08c29", // 자격 증명 풀ID
  }),
});

const changeProfile = (e) => {
    const image = e.target.files[0];
    if (!image) {
      return setProfile(null);
    }

    const upload = new AWS.S3.ManagedUpload({
      params: {
        Bucket: "subllet-profile",
        Key: image.name,
        Body: image,
      },
    });

    const promise = upload.promise();

    promise.then(
      (data) => {
        setProfile(data.Location);
      },
      (err) => {
        console.log(err);
      }
    );
  };

//...

<label htmlFor="file">프로필 사진변경</label>
<input
	type="file"
  id="file"
  style={{ display: "none" }}
  onChange={changeProfile}
/>
```

위처럼 코드를 작성하면 S3 버킷에 이미지가 저장된다!!
