### 오늘 한것

- 회원삭제 페이지의 CSS와 대략적인 틀을 구성하였다.
- 로그인이 되지 않았을때 띄울 로그인 요청 모달을 만들었다.
- 따로 api를 요청해서 데이터를 받아올 수 없기 때문에 프로젝트에 사용될 데이터를 검색하고 정리하였다.

### 힘든 점

- media 쿼리를 사용해 반응형으로 페이지를 구성하는게 어려웠다. 다른 사이트의 페이지처럼 모바일에도 대응할 수 있도록 최소넓이 320px을 맞추고 하는것이 다른 컴포넌트와 겹치다보니 확대되거나 이상하게 렌더링 되는것을 확인하였다. media 쿼리와 반응형 페이지에 대해서 공부가 더 필요한것 같다.
- 로그인 요청 모달창을 만들 때 모달을 띄우는것 까지는 구현 했지만 모달창 밖을 클릭했을 때 없어지는 구현이 조금 어려웠다. 해결방법으로는 모달 컴포넌트에서 ModalBack div태그를 만들어 넓이와 높이를 100%를 주고 onclick으로 open 상태를 변경시켜주는 함수를 걸어주었다.

```jsx
const LoginModal = ({ handleClick }) => {
  return (
    <>
      <ModalBack onClick={handleClick} />
      <StyledBody>
        <div>로그인 하신 후 이용해 주시기 바랍니다.</div>
        <Link to="/userlogin">
          <button>로그인</button>
        </Link>
      </StyledBody>
    </>
  );
};
```

- 데이터를 하나하나 다 찾는것이 생각보다 어려웠다.. 상세보기페이지의 이미지와 로고이미지를 다 찾아야하고 서비스들을 찾아봐야하는게 은근 양이 많았다.. 이래서 api를 사용하는구나 싶기도 했다 일단 내일마저 더 찾아보고 이미지를 s3로 할것인지 데이터베이스에 할것인지를 팀원들과 한번더 상의 후 정리해야겠다.
