### 오늘 한것

- 상세보기 페이지를 redux를 최상위에서 사용하기로 했다가 오류가 있어서 다시 해당 페이지에서 axios요청을 보내는 코드로 수정하였다.

### 이유

- redux를 사용하기로 했는데 막상 보니 한 페이지당 1단계의 자식 컴포넌트들 밖에 없었고 1단계에서의 상태 공유면 props로 바로 내려주는것이 더 쉽지 않을까? 생각이 들었다 redux를 사용하는 이유는 크게 많은 컴포넌트들로 props 사용이 불편하고 상태 관리가 힘들어서 사용한다고 이해하고 있었지만 아직은 그리 크지 않은 프로젝트이다 보니 1단계씩밖에 없어서 그런건가.. 큰 프로젝트에서 redux를 사용하나? 이런 의문이 남아있었다
- 그런상황에서 각 페이지에서 필요한 데이터를 얻기 위해 axios로 요청을 보냈고 각 페이지마다 겹치는 api 요청들이 있었다 그렇게 팀원들과 상의 후 최상위인 app.js에서 axios 요청을 보내고 각 페이지나 컴포넌트에서 필요한 데이터가 있거나 변경사항이 있으면 redux로 관리하면 되지 않을까? 라는 생각이 들었고 코드를 수정하기로 결정했다
- 하지만 그렇게 하다보니 몇가지 문제점들이 있었다.. 나의 예로는 상세보기 페이지에서 필요한 데이터를 불러오려 했을 때 typeerror가 등장했다.. 다시 보니 useEffect로 axios통신을 요청하는것 보다 먼저 코드가 실행되었고 그 때 dispatch가 되기 전에 해당 데이터를 불러와서 undefined인 상태이기 때문에 에러가 나는걸로 확인되었다..
