## @import

- 1개의 CSS 파일에 모든 스타일을 작성하면 유지보수하기 힘들고 가독성이 좋지 않은 단점에 따라 CSS 파일을 분리해서 사용하면 재사용 및 유지보수 측면에 유리하다.
- SASS는 @import를 사용하여 분리된 stylesheet 파일을 import할 수 있다.

```jsx
@import "foo.scss";
@import "foo";
// 확장자는 생략 가능

@import "nav", "footer";
// 여러개의 파일을 한번에 임포트할 수 있다.
```

## @extend

- 기존 스타일을 상속할때 사용한다. (확장)

```jsx
// 예시
.nav {
    padding: 10px;
    margin: 10px;
    background: red;
}
.nav-blue {
    @extend .nav;
    background: blue;
}

// 컴파일 후
.nav .nav-blue {
    padding: 10px;
    margin: 10px;
    background: red;
}
.nav-blue {
    background: blue;
}
```

- 위와 같이 다중 선택자를 만들 수 있다.
- 하지만 확장 기능은 부작용이 있을 수 있어 Mixin 사용을 권장한다.


## Mixin

- Mixin은 재사용할 CSS 선언 그룹을 정의하는 기능이다.
- `@mixin`을 선언하고 `@include`로 불러들인다.

```jsx
@mixin name {
  style;
}

// 예시
@mixing square {
    width: 20px;
    height: 20px;
}

.box {
    @include square;
    // 추가적인 스타일지정도 가능
    background: red;
}
```

- 위에 예시처럼 추가적인 스타일 지정도 가능하다.

```jsx
// 예시
@mixin square($size) {
    width: $size;
    height: $size * 2;
}

.box {
    @include square(10px);
}
```

- mixin은 함수와 같이 매개 변수를 사용할 수 있다.

```jsx
// 예시
@mixin square($size: 10px) {
    width: $size;
    height: $size * 2;
}

.box {
    @include square();
}
```

- 매개 변수의 초기값 설정도 가능하다.
