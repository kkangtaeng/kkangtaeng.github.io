아직 태그를 사용하는 법이나 정확한 의미는 모르지만 앞으로 참고할일을 대비해 자주 사용하는 태그들만 찾아서 정리해보려고 한다!!

## 자주 사용되는 태그(Tag)

### `<a>` 태그

- 다른 컨텐츠와 연결되는 하이퍼링크를 정의하는 태그
- href 속성으로 링크를 넣을 수 있다

```jsx
//ex
<a href="https://kkangtaeng.github.io">이동!!</a>
```

### `<br>` 태그

- 텍스트 내의 줄바꿈을 정의하는 태그
- 닫는 태그없이 사용한다

### `<button>` 태그

- 클릭할 수 있는 버튼을 정의하는 태그

### `<div>` 태그

- 컨텐츠를 하나로 묶어주기 위해 사용하는 태그
- 줄바꿈이 된다 (block-element)

### `<h1> ~ <h6>` 태그

- 문서에서 제목을 정의하는 태그
- `<h1>`태그가 글자가 제일 크고, 갈수록 크기가 작아진다

### `<head>` 태그

- 웹 페이지의 정보나 문서에서 사용하는 외부 파일들을 링크할 때 사용하는 태그

### `<html>` 태그

- 웹 페이지의 시작과 끝을 정의하는 태그

### `<img>` 태그

- 웹 페이지에 이미지를 넣을 때 사용하는 태그
- `<img>`태그 하나당 1개의 이미지만 삽입 가능
- src 속성과 alt 속성을 반드시 넣어줘야 한다

```jsx
//ex
<img src="images/emoji" alt="이모지">
```

### `<input>` 태그

- 사용자로부터 입력을 받을 수 있는 필드를 정의하는 태그

### `<li>` 태그

- `<ul>`태그와 `<ol>`태그 안에서 각 항목을 나열할 때 사용하는 태그
- 목록 아이템을 나타낸다

### `<link>` 태그

- 외부 파일을 연결할 때 사용하는 태그

### `<ol>` 태그

- 순서가 있는 리스트를 정의하는 태그

### `<p>` 태그

- 문단을 정의하는 태그
- 내용 앞뒤로 빈줄이 생긴다

### `<script>` 태그

- 자바스크립트와 같은 클라이언트 사이드 스크립트를 정의하는 태그

### `<span>` 태그

- 컨텐츠를 하나로 묶어주기 위해 사용하는 태그
- 줄바꿈이 되지 않는다 (inline-element)

### `<title>` 태그

- 문서의 제목을 정의하는 태그

### `<ul>` 태그

- 순서가 없는 리스트를 정의하는 태그

## 시멘틱 태그(Tag)

- 특정한 태그에 의미를 부여한 태그

### `<article>` 태그

- 해당 문서나 페이지 또는 사의트와는 독립적으로 구성할 수 있는 요소를 정의하는 태그
- 시각적인 효과를 주는 태그는 아니지만, 본문을 명시해주기 위해 사용

### `<asdie>` 태그

- 해당 컨텐츠들로부터 분리시킬 수 있는 컨텐츠로 구성된 페이지 영역을 정의하는 태그
- 사이드에 위치하는 공간을 의미

### `<footer>` 태그

- 문서나 특정 섹션의 푸터를 정의하는 태그
- 저작권 정보나 제작 정보 등을 표시한다

### `<header>` 태그

- 문서나 특정 섹션의 헤더를 정의하는 태그

### `<nav>` 태그

- 다른 페이지 또는 현재 페이지의 다른 부분과 연결하는 링크들을 정의하는 태그

### `<section>` 태그

- 독립적인 섹션을 정의하는 태그
