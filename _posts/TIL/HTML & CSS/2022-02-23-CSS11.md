## Sass

- CSS 전처리기로 CSS의 한계와 단점을 보완하여 보다 가독성이 높고 코드의 재사용에 유리한 확장이다.
- 프로젝트의 규모가 커지고 수정이 빈번히 발생함에 따라 유지보수를 용이하게 하기위해 사용한다.

### SASS의 장점

1. CSS보다 심플한 표기법으로 CSS를 구조화하여 표현 가능하다.
2. 스킬 레벨이 다른 팀원들과의 작업 시 발생할 수 있는 구문의 수준 차이를 평준화할 수 있다.
3. CSS에는 존재하지 않는 기능을 활용하여 유지보수에 용이하다.

### 데이터 타입

- Sass는 속성값으로 7가지의 데이터 타입을 제공한다.
  1. Number
  2. String
  3. Color
  4. boolean
  5. null
  6. list
  7. map

### 변수 선언

- Sass에서는 변수 선언이 가능하고 문자열, 숫자, 색상 등을 사전에 변수에 저장하고 불러올 수 있다.
- 변수명은 달러($)를 먼저 작성한다.

```jsx
$variable-name: variable-value;

// 예시
$nav-color: red;
```

## 연산자

### 숫자 연산자

- Sass는 연산자를 사용할 수 있지만 상대적인 값은 알지 못해서 상대적인 값을 갖는 단위의 연산은 동일한 단위를 갖는 값과의 연산만이 가능하다. (calc 함수를 사용하면 가능)

|연산자|설명|
|---|---|
|+|덧셈|
|-|뺄셈|
|*|곱셈|
|/|나눗셈|
|%|나머지|
|==|동등|
|!=|부등|

- CSS에서 `/`는 나눗셈이 아니라 값을 구분하는 의미로 사용되어 나눗셈 연산자를 사용하기 위해서는 3가지 조건을 사용해야 한다.
  1. 변수에 대해 사용

  ```jsx
  // 예시
  $width: 100px;

  width: $width / 2;
  ```

  2. 괄호 안에서 사용

  ```jsx
  // 예시
  width: (100px / 2);
  ```

  3. 다른 연산의 일부로서 사용

  ```jsx
  // 예시
  width: 100px + 100px / 2px;
  ```

- 변수를 함께 사용하고자 할 경우에는 `#{}`를 사용한다.

```jsx
// 예시
div {
  $font-size: 12px;
  $line-height: 30px;
  font: #{$font-size}/#{$line-height};  // 12px/30px
}
```

- 숫자 연산자로 색상 연산도 할 수 있다.
- 문자열도 Javascript와 같이 연산하여 연결할 수 있다.
- boolean 연산자도 사용할 수 있다.(&&, `||`, !)
