## 사용권한

- 명령어 `ls -l`을 입력하면 파일의 권한을 알 수 있다.

![다운로드](https://user-images.githubusercontent.com/85768164/150669433-85e2d9bf-fd83-411f-bb01-1796dbdea5d5.png)

처음 시작은 `-`와 `d`로 각각 not directory와 directory로 파일이면 `-` 폴더이면 `d`를 나타낸다.  
다음으로 `r`은 read permission으로 일기 권한,  
`w`는 write permission으로 쓰기 권한,  
`x`는 execute permission으로 실행 권한을 나타낸다.

- user: 파일의 소유자로 기본적으로 파일을 만든 사람이 소유자가 된다. 따라서 user를 소유자라고 하기도 한다.
- group: 여러 user가 포함될 수 있는데, 그룹에 속한 모든 user는 파일에 대한 동일한 group 엑세스 권한을 갖는다.
- other: 파일에 대한 엑세스 권한이 있는 다른 user이며, 파일을 만들지 않은 다른 모든 user를 의미한다.

## chmod

- 폴더나 파일의 권한을 변경하는 명령어로, 읽기, 쓰기, 실행 권한을 변경할 수 있다.
- 폴더나 파일의 소유자가 다를 경우엔, `sudo`명령어를 통해 변경할 수 있다.

### Symbolic method

- `+`, `-`, `=`과 액세스 유형을 표기해서 변경하는 방법이다.
- Access class, Operator, Access Type 으로 구분한다.

|Access class|Operator|Access Type|
|---|---|---|
|u (user)|+ (add access)|r (read)|
|g (group)|- (remove access)|w (write)|
|o (other)|= (set exact access)|x (execute)|
|a (all)||

```jsx
// 예시
chmod g-r test.js
chmod g+w test.js
```

### Absolute form

- 숫자 7까지 나타내는 3 bit의 합으로 표기하여 변경하는 방법이다.
- 각 그룹마다 rwx가 나타나고, 각 영역의 boolean 값으로 표기할 수 있다.

|Permission|Number|
|---|---|
|r (read)|4|
|w (write)|2|
|e (execute)|1|

```jsx
//예시
chmod 744 test.js // -rwxr--r--
// user=rwx(4+2+1), group/read=r(4+0+0)
```

## 환경변수

- 명령어 `export`를 이용해, 기록된 환경변수를 확인할 수 있다.
- 명령어 `export`를 이용해, 새로운 환경변수를 추가할 수 있다.
```jsx
export urclass="is good"
// 이 때 등호 표시 앞뒤에는 반드시 공백이 없어야 한다!!
```
- 명령어 `echo`와 환경변수를 같이 입력하면, 환경변수의 값을 확인할 수 있다. 이 때 환경변수의 앞에는 `$`을 입력해야 한다.
```jsx
echo $urclass
// is good
```

### dotenv

- dotenv를 이용하여 Javascript에서 환경변수를 사용할 수 있다.
```jsx
mkdir environment_variable
cd environment_variable
npm init // 엔터 키를 여러번 입력한다.
npm i dotenv // dotenv 모듈을 설치합니다.
// 이 때 생성된 폴더는 설치할 모듈과 이름이 같으면 안된다.
```

### .env

- 환경변수를 Node.js에서 영구 적용하는 방법이다.
```jsx
const dotenv = require("dotenv");
dotenv.config();
console.log(process.env.myname);
```
index.js를 위처럼 수정하고, node index.js를 하면 환경변수를 출력할 수 있다.
