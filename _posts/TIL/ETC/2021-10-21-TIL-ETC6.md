## 토큰 (Token)

### 토큰 기반 인증 (Token-based Authentication)

- 세션 기반 인증에서 매 요청마다 데이터베이스를 살펴보는 부담을 덜어내기 위해 클리이언트에서 인증 정보를 보관하는 방법으로 토큰 기반 인증이 고안되었다.

### JWT (JSON Web Token)

- 클라이언트에서 XSS, CSRF 공격에 노출이 될 위험으로 보호하기 위해 JSON 포맷으로 사용자에 대한 속성을 저장하는 웹 토큰이다.

### JWT의 종류

1. Access Token
  - Acess Token은 보호된 정보들(유저의 이메일, 연락처, 사진 등)에 접근할 수 있는 권한부여에 사용된다.
  - 처음 인증을 받게 될 때(로그인 등) Access Token, Refresh Token 두가지를 받지만, 권한을 얻는데 사용하는 토큰은 Access Token 이다.
  - 그로인해 Access Token은 비교적 짧은 유효 기간을 주어 탈취되더라도 오랫동안 사용할 수 없도록 하는것이 좋다.

2. Refresh Token
  - Access Token의 유효기간이 만료되면 새로운 Access Token을 발급받는데에 사용된다.

### JWT의 구조

![qqq](https://user-images.githubusercontent.com/85768164/152507745-ab32b5cf-cf25-4170-a1a8-d3d1e7605cb8.png)

JWT는 위 그림과 같이 `.`으로 나누어진 3부분이 존재한다.

1. Header
  - Header는 어떤 종류의 토큰인지, 어떤 알고리즘으로 sign할지가 적혀있다.
  ```jsx
  // 예시
  {
      "alg": "HS256",
      "typ": "JWT"
  }
  ```

2. Payload
  - 어떤 정보에 접근 가능한지에 대한 권한을 담을 수도 있고, 사용자의 유저 이름 등 필요한 데이터를 담아 Sign 시키므로, 민간한 정보는 되도록 담지 않는 것이 좋다.
  ```jsx
  // 예시
  {
      "sub": "someInformation",
      "name": "phillip",
      "iat": 151623391
  }
  ```

3. Signature
  - 원하는 비밀키(암호화에 추가할 salt)를 사용하여 암호화 한다.
  ```jsx
  // HMAC SHA256 알고리즘을 사용한 예시
  HMACSHA256(base64UrlEncode(header) + '.' + base64UrlEncode(payload), secret);
  ```

### 토큰 기반 인증 절차

![www](https://user-images.githubusercontent.com/85768164/152508999-828606fa-c841-4a39-9048-822c6bb420c3.jpeg)

1. 클라이언트가 서버에 아이디, 비밀번호를 담아 로그인 요청을 보낸다.
2. 아이디, 비밀번호가 일치하는지 확인하고, 클라이언트에게 보낼 Signature된 토큰을 생성한다.
  - Access Token, Refresh Token 을 모두 생성하며, 토큰에 담길 payload는 유저를 식별할 정보, 권한이 부여된 카테고리 등이 될 수 있다.
3. 토큰을 클라이언트에게 보내주면, 클라이언트는 토큰을 저장한다.
4. 클라이언트가 HTTP Header에 토큰을 담아 보낸다.
5. 서버는 토큰을 해독하여 일치할 경우, 클라이언트의 요청을 처리하고 응답을 보내준다.

### 토큰 기반 인증의 장점

1. Statelessness & Scalability (무상태성 & 확장성)
  - 서버는 클라이언트에 대한 정보를 저장할 필요가 없고 오로지 토큰 해독이 되는지만 판단한다.
  - 클라이언트는 새로운 요청을 보낼 때마다 토큰을 헤더에 포함시키면 되므로, 서버를 여러개 가지고 있는 서비스라면 확장성이 매우 좋다.
2. Signature를 받아 토큰을 사용하고, 암호화 키를 노출할 필요가 없기 때문에 안전하다.
3. 토큰을 확인하는 서버가 토큰을 만들어야 하는 법이 없고, 토큰 생성용 서버를 만들거나, 다른회사에서 토큰 관련 작업을 맡기는 등 다양한 활용이 가능하다.
4. 토큰의 payload안에 어떤 정보에 접근 가능한지 정할 수 있으므로, 권한 부여에 용이하다.
