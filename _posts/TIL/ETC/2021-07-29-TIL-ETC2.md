## Git 명령어

<img width="515" alt="git" src="https://user-images.githubusercontent.com/85768164/147481509-10f71747-877b-4310-9da6-7216bc8cc0f5.png">

### Fork

- 깃허브 우측 상단에 Fork 버튼을 클릭하여 유저 네임을 클릭
- 다른 계정의 remote repository를 자신 계정의 repository로 가지고 올 때 사용
- Fork 한 시점부터 프로젝트가 자신의 repository에 복사본으로 저장됨

### git clone

```jsx
git clone <repository 주소>
```

- 원격 repository를 내 local에서 이용할 수 있도록 복사할 때 사용
- 자신의 remote repository에 있는 파일을 작업하기 위해 내 컴퓨터로 복사해 오는 명령어

### git add

```jsx
git add <파일명> // 내 local의 untracked file을 staging area로 추가

git add . // staging area에 모든 파일을 한번에 추가
```

- untracked files를 staging area로 추가해서 git의 관리하에 둠
- commit 하기 이전에 먼저 git의 관리 하인 staging area로 파일을 옮겨줘야 함.

### git commit

```jsx
git commit

git commit -m '메세지' // -m 옵션을 통해 커밋할 내용의 코멘트를 작성할 수 있음.
```

- 수정 작업이 끝났을 때 변경 사항을 저장
- staging area에 있는 파일을 local repository로 옮김.
- -m 옵션을 통해 commit을 하면 날짜, commit한 사람, commit 메세지가 모두 기록됨.

### git push

```jsx
git push <origin> <branch>
```

- local에서 변경, commit 된 사항을 remote repository에 업로드
- git push 뒤에 따라오는 명령어는 상황에 따라 변경할 수 있음

### git pull

```jsx
git pull <이름> <branch>
```

- remote repository의 작업 내용 다운로드
- 상대방이 작업한 코드를 받아올 수 있음
- 다운로드된 내용은 자동으로 병합(merge) 됨

### git init

```jsx
git init
```

- 기존 디렉토리를 git repository로 변환
- 내 컴퓨터에만 존재하는 디렉토리를 git 관리 하에 들어가게 만들어 주기 위해 사용
- 새로운 repository를 초기화 하는데에도 사용

### git restore

```jsx
git restore <파일명> // add되지 않은 변경 내용을 초기화

git restore --staged <파일명> // commit 되지 않은 변경 사항을 삭제
```

- commit 이전 local repository의 변경사항을 취소

### git reset

```jsx
	git reset HEAD^ or HEAD~1 // local 에서 commit 한 내용을 취소
```

- 아직 remote repository에 올라가지 않은 commit 을 취소

### remote add origin

```jsx
git remote add origin <repository 주소>

// local repository에 remote repository 연결
```

- 나의 remote repository에 연결할 때 사용

### remote add pair

```jsx
git remote add <이름> <repository 주소>

// local repository에 remote repository 연결
```

- 다른 사람의 remote repository에 연결할 때 사용

### git status

```jsx
git status // 내 local의 staging area, untracked files 목록을 확인할 수 있음
```

- 자신의 local로 복사해 온 디렉토리의 commit 되기 전 까지의 상태를 표시

### git log

```jsx
git log
```

- 현재까지 commit된 내역들을 터미널 창에서 표시

### remote -v

```jsx
git remote -v //연결된 remote repositorty 확인
```

- 현재의 local repository와 연결된 모든 remote repository 목록을 표시

### git branch

```jsx
git branch <새로운 branch 이름>

git branch
```

- 새로운 브랜치를 생성한다
- 브랜치 이름 없이 입력하면 브랜치 목록을 확인할 수 있다

### git branch -v

```jsx
git branch -v
```

- 브랜치 목록과 각 브랜치의 최근 커밋을 확인할 수 있다

### git branch -d

```jsx
git branch -d <삭제할 branch 이름>
```

- 해당 브랜치를 삭제 한다

### git switch / git checkout

```jsx
git switch <branch 이름>

git checkout <branch 이름>
```

- 해당 브랜치로 이동한다

### git switch -c / git checkout -b

```jsx
git switch -c <새로운 branch 이름>

git checkout -b <새로운 branch 이름>
```

- 새로운 브랜치를 생성하여 해당 브랜치로 이동한다

### git merge

```jsx
git merge <병합할 branch 이름>

// ex)
// master branch로 dev 브랜치를 병합할 때
git switch master
// master 브랜치로 이동한다
git merge dev
// master 브랜치로 dev 브랜치를 병합한다
```

- 해당 브랜치를 현재 위치의 브랜치와 병합한다
