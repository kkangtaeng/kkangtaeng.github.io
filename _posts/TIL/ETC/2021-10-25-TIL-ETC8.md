## 문자열

### 유니코드 (Unicode)

- 유니코드(Unicode)는 유니코드 협회가 제정하는 전 세계의 모든 문자를 컴퓨터에서 일관되게 표한하고 다룰 수 있도록 설계된 산업 표준이다. 이 표준에는 ISO 10646 문자 집합, 문자 인코딩, 문자 정보 데이터베이스, 문자를 다루기 위한 알고리즘 등을 포함하고 있다.

### ASCII 문자

- 영문 알파벳을 사용하는 대표적인 문자 인코딩이다.
- 7bit로 모든 영어 알파벳을 표현할 수 있고, 52개의 영문 알파벳 대소문자와, 10개의 숫자, 32개의 특수문자, 그리고 하나의 공백 문자를 포함한다.
- 유니코드는 ASCII를 확장한 형태이다.

## UTF-8과 UTF-16

- UTF-8과 UTF-16은 인코딩 방식의 차이를 말하며, Universal Coded Character Set + Transformation Format + bit의 약자로, UTF- 뒤에 등장하는 숫자는 bit이다.

### UTF-8의 특징

- UTF-8은 유니코드 한 문자를 나타내기 위해 1 byte(= 8 bits)에서 4 bytes까지의 가변길이를 가지는 인코딩 방식으로, 네트워크를 통해 전송되는 텍스트는 주로 UTF-8로 인코딩 한다.
- UTF-16에 비해 바이트 순서를 따지지 않고, 순서가 정해져 있다.

### UTF-16의 특징

- 유니코드 코드 대부분을 16 bits로 표현한다.
- byte 순서가 다양하다.

## 운영체제

### 운영체제가 하는 일

1. 시스템 자원 관리

- 프로세스 관리(CPU)
- 메모리 관리
- I/O(입출력) 관리 (디스크, 네트워크 등)

2. 응용 프로그램 관리

- 응용 프로그램이 실행되고, 시스템 자원을 사용할 수 있도록 권한과 사용자를 관리

### 프로세스

- 운영체제에서 실행중인 하나의 어플리케이션을 말한다. 즉, 사용자가 어플리케이션을 실행할 때 운영체제로부터 실행에 필요한 메모리를 할당받아 어플리케이션의 코드를 실행하는데 이것을 프로세스라고 한다.
- 예를들어 크롬 브라우저 2개를 실행하면 두개의 프로세스가 생성되고 이렇게 하나의 어플리케이션은 여러 프로세스를 만들기도 한다.

### 스레드 (Thread)

- 하나의 스레드는 코드가 실행되는 하나의 흐름이기 때문에, 한 프로세스 내에 스레드가 두개라면 코드가 실행되는 흐름이 두개 생긴다는 뜻이다.

### 스레드의 특징

- 프로세스 내에서 실행되는 흐름의 단위
- 각 스레드마다 call stack이 존재 (call stack: 실행중인 서브루틴을 저장하는 자료구조)
- 스레드는 다른 스레드와 독립적으로 동작

### 멀티 스레드 (Multi-Thread)

- 어플리케이션 내부에서 2가지 이상의 작업을 동시에 처리하는 것을 말한다.

### 멀티 스레딩의 장점

- 프로세스를 이용하여 동시에 처리하던 일을 스레드로 구현할 경우, 메모리 공간과 시스템 자원의 소모가 줄어들고 시스템 처리량이 향상되고 자원 소모가 줄어들어 프로그램의 응답 시간이 단축된다.
- 스레드 간의 통신이 필요한 경우에도 별도의 자원을 이용하는 것이 아니라, 전역 변수의 공간 또는 동적으로 할당된 공간인 Heap 영역을 이용하여 프로세스 간 통신 방법에 비해 스레드 간의 통신 방법이 훨씬 간단하다.

### 멀티 스레딩의 문제점

- 서로 다른 스레드가 같은 데이터에 접근하고, Heap 영역을 공유하기 때문에 서로 다른 스레드가 서로 사용중인 변수나 자료구조에 접근하여 엉뚱한 값을 읽어오거나 수정하는 일이 발생할 수 있다.
- 그러므로 멀티스레딩 환경에서는 동기화 작업이 필요하고, 동기화 작업을 통해 작업 처리 순서를 제어하고, 공유 자원에 대한 접근을 제어해야한다.

### 동시성과 병렬성의 차이

- Concurrency (동시성, 병행성): 여러 개의 스레드가 시분할 방식으로 동시에 수행되는 것처럼 착각을 불러일으킨다.
- Parallelism (병렬성): 멀티 코어 환경에서 여러 개의 스레드가 실제로 동시에 수행된다.
