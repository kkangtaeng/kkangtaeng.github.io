## ORM

- Javascript의 객체와 클래스를 ORM을 통해 데이터베이스의 엔티티, 레코드 들을 다루는 일을 한다. 즉, Javascript의 객체와 관계형 데이터베이스의 중개자 역활을 한다.

## Sequelize

- Promise기반의 Node.js ORM이다.

### Sequelize 설치

```jsx
npm install --save sequelize
```

### Sequelize-cli 설치

- Migration을 할 수 있도록 돕는 툴로, CLI에서 Model을 생성해 주거나, 스키마 적용을 할 수 있도록 한다.
- `npx sequelize-cli --help`를 통해 명령의 종류를 파악할 수 있다.

```jsx
npm install --save-dev sequelize-cli
```

### bootstraping

- 먼저 CLI를 통해 ORM을 잘 사용할 수 있도록 bootstraping(프로젝트 초기 단계를 자동으로 설정할 수 있도록 도와주는 일)을 해야한다.

```jsx
config/config.json
models/
migrations/
seeders/
// 성공적으로 bootstraping이 완료되면 다음 파일 및 폴더들이 생성된다.
// 직접 models 폴더를 만드는것은 아니다!
```

### Model 생성

- Model을 만들기 위해서 `model:generate`라는 명령어를 사용하는데 2가지 옵션이 있다.
  - `name`: Model의 이름
  - `attributes`: 모델의 속성 목록

```jsx
// User라는 이름의 Model 생성 예시
npx sequelize-cli model:generate --name User --attributes firstName:string,lastName:string,email:string
```
- `firstName, lastName, email` 필드는 자동으로 생성되며, `models` 폴더 안에 `user`라는 파일의 model이 생성되고 `migrations` 폴더 안에 `XXXXXXXXXXXXXX-create-user.js`와 같은 파일이 생성된다.

- Model을 잘못 만든 경우, 생성된 파일을 직접 수정하거나 삭제 후 명령을 다시 실행할 수 있다.

### Migration

- 이제 데이터베이스에 테이블을 생성하려면 `db:migrate` 명령어를 입력해야 한다.

```jsx
npx sequelize-cli db:migrate
```
