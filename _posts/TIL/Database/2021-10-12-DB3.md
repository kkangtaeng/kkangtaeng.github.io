## 관계형 데이터베이스

- 구조화된 데이터는 하나의 테이블로 표현할 수 있고, 사전에 정의된 테이블을 사용하는 데이터베이스 이다.

### 관계형 데이터베이스의 키워드

- 데이터 (data) - 각 항목에 저장되는 값
- 테이블 (table) - 사전에 정의된 열의 데이터 타입대로 작성된 데이터가 행으로 축적되는 표
- 칼럼, 열 (column, field) - 테이블의 열
- 레코드, 행 (record, tuple) - 테이블의 행
- 키(key) - 테이블의 각 레코드를 구분하는 값
- 기본키 (primary key) - 테이블의 고유한 값
- 외래키 (foreign key) - 다른 테이블의 기본키를 참조하는 키

### 관계 종류

- 1 : 1 - 하나의 레코드가 다른 테이블의 레코드 한개와 연결된 경우
- 1 : N - 하나의 레코드가 서로 다른 여러 개의 레코드와 연결된 경우
- N : N - 여러 개의 레코드가 다른 테이블의 여러 개의 레코드와 관계가 있는 경우

## 집합연산

- 레코드를 조회하고 분류한 뒤, 특정 작업을 하는 연산이다.

### GROUP BY

- 데이터를 조회할 때 그룹으로 묶어서 조회한다.

```jsx
SELECT * FROM 테이블명
GROUP BY 특정값

// 특정값에 따라 데이터를 그룹화 한다.
```

### HAVING

- GROUP BY 로 조회된 결과를 필터링한다.

```jsx
SELECT * FROM 테이블명
GROUP BY 특정값
HAVING 조건

// 특정값에 따라 데이터를 그룹화 한뒤 조건에 따라 필터링한다.
```

## SQL 내장 함수

### COUNT()

- 레코드의 개수를 센다.

```jsx
SELECT COUNT(레코드명)
FROM 테이블명
WHERE 조건
```

### AVG()

- 레코드의 평균을 낸다.

```jsx
SELECT AVG(레코드명)
FROM 테이블명
WHERE 조건
```

### SUM()

- 레코드의 합을 낸다.

```jsx
SELECT SUM(레코드명)
FROM 테이블명
WHERE 조건
```

### MIN()

- 레코드의 최솟값을 낸다.

```jsx
SELECT MIN(레코드명)
FROM 테이블명
WHERE 조건
```

### MAX()

- 레코드의 최대값을 낸다.

```jsx
SELECT MAX(레코드명)
FROM 테이블명
WHERE 조건
```

### SELECT 실행 순서

1. FROM
2. WHERE
3. GROUP BY
4. HAVING
5. SELECT
6. ORDER BY
