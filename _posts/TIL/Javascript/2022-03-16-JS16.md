## TypeScript를 사용하는 이유

Javascript는 타입 시스템이 없는 동적 프로그래밍 언어로, 변수에 문자열, 숫자, 불린 등 여러 타입의 값을 가질 수 있다.  
그로인해 유연하게 개발할 수 있는 환경을 제공하는 장점이 있지만, 런타임 환경에서 쉽게 에러가 발생할 수 있는 단점이 있다.

```jsx
// 예시를 한번 보자
// Javascript
function add(num1, num2) {
  console.log(num1 + num2);
}

add(); // NaN
add(1); // NaN
add(1, 2); // 3
add(1, 2, 3); // 3
add("hello", "world"); // 'helloworld'

// TypeScript
function add(num1: number, num2: number) {
  console.log(num1 + num2);
}

add(); // err
add(1); // err
add(1, 2); // 3
add(1, 2, 3); // err
add("hello", "world"); // err
```

위처럼 Javascript는 원하는 방식과 원하는 값을 얻을 수 없고 실수가 있었음에도 아무런 경고없이 실행되게된다.  
Javascript는 동적 프로그래밍 언어로 실행되는 시점 즉, 런타임에 타입이 결정되며 오류가 있으면 런타임시 발견된다.  
반면 TypeScript는 정적 프로그래밍 언어로 컴파일 타임에 타입이 결정되며 오류가 있으면 발견된다. 코드 작성시에 코드를 잘 짜여진다면 안정적이고 빠르게 작업을 할 수 있는 장점이 있다.

## 기본 타입

- 프로그램이 유용하려면 문자열, 숫자, 불리언 등과 같은 간단한 데이터 단위가 필요하다.  
  TypeScript는 Javascript와 거의 동일한 데이터 타입을 지원하며, 열거 타입을 사용하여 더 편리하게 사용할 수 있다.

### 작성법

- TypeScript는 일반 변수, 파라미터, 프로퍼티 등에 `: TYPE`과 같은 형태로 타입을 작성한다.

```jsx
// 예시
function add(a: number, b: number) {
  console.log(a + b);
}
```

### 문자열 (String)

- TypeScript도 Javascript처럼 `string`이라고 표현하며 큰따옴표("")나 작은따옴표('')를 사용하여 문자열 데이터를 감싸는데 사용한다.
- 백틱(``)을 사용해 템플릿 문자열도 사용 가능하다.

```jsx
// 예시
let color: string = "black";

let name: string = `kim`;
let age: number = 29;
let introduce: string = `Hello, my name is ${name}. I'm ${age} years old.`;
```

### 숫자 (Number)

- Javascript와 같이 `number`라고 표현한다.

```jsx
let age: number = 29;
```

### 불리언 (Boolean)

- Javascript와 같이 `boolean`이라고 표현한다.

```jsx
// 예시
let isDone: boolean = true;
```

### 배열 (Array)

- Javascript와 같이 값들을 배열로 다룰 수 있으며 두가지 방법으로 사용할 수 있다.

1. []

   ```jsx
   // 예시
   let arr: number[] = [1, 2, 3];
   ```

2. Array<type>

   ```jsx
   // 예시
   let arr: Array<number> = [1, 2, 3];
   ```

### 튜플 (Tuple)

- 튜플 타입은 요소의 타입과 개수가 고정된 배열을 표현할 수 있다. 즉, 인덱스 별로 타입이 다를 때 사용할 수 있으며 타입이 모두 같을 필요는 없다.

```jsx
// 예시
let a: [string, number];

a = ["Hello", 1]; // ok
a = [1, "Hello"]; // err

a[0].toLowerCase(); // 'hello'
a[1].toLowerCase(); // err
```

### 열거 (Enum)

- enum은 숫자 혹은 문자열 값 집합에 이름을 부여할 수 있는 타입으로, 값의 종류가 일정한 범위로 정해져 있는 경우 유용하다.

```jsx
// 예시
enum Color {
    Red, // 0
    Blue, // 1
    Black // 2
}
// 기본적으로 0부터 시작하며 값은 1씩 증가한다.

enum Color {
    Red, // 0
    Blue = 5, // 5
    Black // 6
}
// 수동으로 값을 변경할 수 있으며, 값을 변경한 부분부터 다시 1씩 증가한다.
```

### 모든 타입 (Any)

- Any는 모든 타입을 의미한다. 따라서 일반적인 Javascript 변수와 동일하게 어떤 타입의 값도 할당할 수 있으며 외부 자원을 활용해 개발할 때 불가피하게 타입을 단언할 수 없는 경우에 유용하다.

```jsx
let some: any = 123; // ok
any = "hello"; // ok
```

### Void

- Void는 일반적으로 값을 반환하지 않는 함수에서 사용한다.

```jsx
// 예시
function hello(name: string): void {
  console.log(`hello ${kim}`);
}
```

### Never

- Never는 절대 발생하지 않을 값을 나타내며, 어떠한 타입도 적용할 수 없다.
- 보통 항상 에러만을 반환하거나 영원히 끝나지 않는 함수의 타입으로 사용할 수 있다.

```jsx
// 예시
function showErr(): never {
  throw new Error();
}

function Loop(): never {
  while (true) {
    // ...
  }
}
```
