### 브라우저의 동작원리

- 브라우저는 웹페이지를 서버에 요청하고 서버의 응답을 받아 표시한다.
- 서버로부터 HTML, CSS, Javascript를 받는데 HTML, CSS 파일은 렌더링 엔진의 HTML 파서와 CSS 파서에 의해 파싱되어 DOM, CSSOM 트리로 변환되고 렌더 트리로 결합된다. 이렇게 생성된 렌더 트리를 기반으로 브라우저는 웹페이지를 표시한다.

### Javascript의 동작원리

- Javascript는 렌더링 엔진이 아닌 Javascript 엔진이 처리하는데 HTML 파서는 script 태그를 만나면 DOM 생성 프로세스를 중지하고 Javascript 엔진으로 script 태그 내의 코드를 로드하고 파싱하여 실행한다. Javascript의 실행이 완료되면 다시 HTML 파서로 제어 권한을 넘겨 브라우저가 중지했던 시점부터 DOM 생성을 재개한다.
- 브라우저는 동기적으로 HTML, CSS, Javascript를 처리하여 script 태그의 위치에 따라 블로킹이 발생하여 DOM의 생성이 지연될수도 있다.
- body 요소의 가장 아래에 Javascript를 위치시키는 것이 좋은 이유
  - HTML 요소들이 스크립트 로딩 지연으로 인해 렌더링에 지장 받는 일이 발생하지 않아 페이지 로딩 시간이 단축된다.
  - DOM이 완성되지 않은 상태에서 Javascript가 DOM을 조작한다면 에러가 발생한다.

### 모든 자바스크립트 파일을 브라우저에서 한 번에 로딩 할 때의 문제점

  - script 태그를 만나면 DOM 생성을 중지하고 scipt 태그 내의 코드를 파싱하는데 만약 모든 Javascript 파일을 한번에 불러온다면 HTML이 화면에 출력되는 것이 늦어지므로 Javascipt 파일이 다 로드 될때까지 아무것도 안떠있는 상황이 발생하며 그로 인해 유저 입장에서는 오류가 난것같은 상황이 발생할 수 있다.
