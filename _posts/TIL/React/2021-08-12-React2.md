## SPA

- 서버로부터 모든 새로운 페이지를 불러오지 않고 페이지 갱신에 필요한 데이터만 받아 그 정보를 기준으로 현재의 페이지를 업데이트하는 웹 어플리케이션이다.

### SPA의 장점

- 전체 페이지가 아니라 필요한 부분의 데이터만 받아서 화면을 업데이트 하므로 빠르게 반응한다.
- 서버에서 요청 받은 데이터만 넘겨주면 되기 때문에 서버 과부하 문제가 현저히 줄어든다.
- 전체 페이지를 렌더링 할 필요가 없기 떄문에 더 나은 유저경험을 제공한다.

### SPA의 단점

- JavaScript 파일의 크기 때문에 첫 화면 로딩시간이 길어진다.
  - 브라우저는 첫 화면 로딩시 HTML 파일을 읽어들인 후 script tag 안에 있는 JavaScript 파일을 다시대부분의 코드는 받아오는 과정을 거치는데 이 때 첫 화면 로딩 시 읽어들인 HTML 파일은 거의 비어있고, JavasScriprt 파일안에 들어있다보니 자연스럽게 파일의 크기가 무거워진다.

- 검색 엔진 최적화가 좋지 않다.
  - 검색 로봇은 자료를 수집할 때 웹 페이지의 URL과 HTML 문서 내의 가종 태그나 링크 등을 분석하는데 SPA는 HTML이 거의 비어있다 보니 검색 로봇이 충분한 자료를 수집하지 못하여 검색 노출이 중요한 웹 애플리케이션은 검색 엔진 최적화에 대한 대응책을 마련해야하고, 앱 안에서 브라우저의 앞으로 가기 / 뒤로가기 등의 상태관리도 해야하기 때문에 개발의 복잡도가 늘어난다.

## React Router

- React SPA에서 Routing을 위해 사용하는 라이브러리 이다.

### Routing

- 다른 주소에 따라 다른 뷰를 보여주는 과정, 즉 "경로에 따라 변경한다" 라는 의미이다.

### React Router의 주요 컴포넌트

- BrowserRouter — Router의 역할을 한다

- Switch / Route — 경로를 매칭하는 역할을 한다

- Link — 경로를 변경하는 역할을 한다

```jsx
import { BrowserRouter, Switch, Route, Link } from "react-router-dom";
// 위 컴포넌트들을 사용하기 위해 React Router 라이브러리에서 불러오는 명령어
```

```jsx
// 사용 예시

import { BrowserRouter, Switch, Route, Link } from "react-router-dom";
// 위 컴포넌트를 불러온다

function App() {
  // 메뉴
  return (
    <BrowserRouter>
      {" "}
      {/* Route 를 사용하기 위한 설정을 진행한다 */}
      <div>
        <nav>
          <ul>
            <li>
              <Link to="/">Home</Link>
              {/* Link 컴포넌트를 이용하여 경로를 연결한다 */}
            </li>
            <li>
              <Link to="/about">MyPage</Link>
            </li>
            <li>
              <Link to="/dashboard">Dashboard</Link>
            </li>
          </ul>
        </nav>

        {/* 주소경로와 우리가 아까 만든 3개의 컴포넌트를 연결한다 */}
        <Switch>
          <Route exact path="/">
            <Home />
          </Route>
          <Route path="/about">
            {" "}
            {/* 경로를 설정하고 */}
            <MyPage /> {/* 컴포넌트를 연결한다 */}
          </Route>
          <Route path="/dashboard">
            <Dashboard />
          </Route>
        </Switch>
      </div>
    </BrowserRouter>
  );
}

function Home() {
  // Home 컴포넌트
  return <h1>Home</h1>;
}

function MyPage() {
  // MyPage 컴포넌트
  return <h1>MyPage</h1>;
}

function Dashboard() {
  // Dashboard 컴포넌트
  return <h1>Dashboard</h1>;
}

export default App;
```
