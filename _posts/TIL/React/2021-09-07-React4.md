## React 데이터 흐름

- React의 개발 방식은 페이지 단위가 아닌, 컴포넌트 단위로 시작한다.  
  먼저 컴포넌트를 만들고, 그 컴포넌트를 이용하여 페이지를 조립하여 만든다.  
  즉, React는 상향식(bottom-up)으로 앱을 만든다.
- 하지만 데이터는 위에서 아래로 하향식(bottom-down)으로 컴포넌트 바깥에서 props를 이용해 데이터를 마치 인자 처럼 전달한다.  
  즉, 데이터를 전달하는 주체는 부모 컴포넌트이다. 이는 단항향 데이터 흐름을 따른다.
- 데이터 중에 변하는 값과, 변하지 않는 값을 구분하여 상태(state)를 두어야 한다.  
  하지만 모든 데이터를 상태로 두면 복잡해지기 때문에 최소화하는 것이 가장 좋다.
- 상태의 위치로는 하나의 상태를 기반으로 두 컴포넌트가 영향을 받는다면 공통 소유 컴포넌트를 찾아 그 곳에 위치해야 한다. 즉, 두 개의 자식 컴포넌트가 하나의 상태에 접근하고자 할 때에는 두 자식의 공통 부모 컴포넌트에 상태(state)를 위치해야 한다.

### 상태(state)의 예시

- 부모로부터 props를 통해 데이터가 전달되면 상태(state)가 아니다.
- 시간이 지나도 데이터가 변하지 않으면 상태(state)가 아니다.
- 컴포넌트 안의 다른 상태(state)나 props를 가지고 계산 가능하면 상태(state)가 아니다.

### 상태(state) 끌어올리기

- 하위 컴포넌트에서 이벤트가 발생하여, 부모의 상태를 바꾸어야 하는 상황일 때, 상태(state) 끌어올리기를 한다.
- 상위 컴포넌트의 "상태를 변경하는 함수"를 하위 컴포넌트로 전달하고, 이 함수를 하위 컴포넌트가 실행하게 되면 부모의 상태(state)를 변경할 수 있다.

```jsx
function ParentComponent() {
  // 부모 컴포넌트
  const [value, setValue] = useState("기본 상태값");

  const handleChangeValue = () => {
    setValue("변경된 값");
  };

  return (
    <div>
      <div>{value}</div>
      <ChildComponent handleChangeValue={handleChangeValue} />
    </div>
  );
}
// 자식 컴포넌트에 props로 handler 함수를 내려준다.

function ChildComponent({ handleChangeValue }) {
  // 자식 컴포넌트
  const handleClick = () => {
    handleChangeValue();
  };

  return <button onClick={handleClick}>값 변경</button>;
}
// 부모 컴포넌트로 받은 props의 handler 함수를 onclick 이벤트에 넣어 클릭시 실행시켜 준다.
```

## React에서 비동기 요청

### Side Effect(부수 효과)

- 함수 내에 어떤 구현이 함수 외부에 영향을 끼치는 경우를 말한다.

```jsx
let foo = "hello";

function bar() {
  foo = "world";
}

bar(); // bar는 Side Effect를 발생시킨다.
```

### Pure Function(순수 함수)

- 오직 함수의 입력만이 함수의 결과에 영향을 주는 함수를 말한다.

### Effect Hook

- useEffect는 컴포넌트 내에서 Side Effect를 실행할 수 있게 하는 Hook이다.  
  React 어플리케이션을 작성할 때, AJAX 요청이나 LocalStorage같은 React와 상관없는 API를 사용하는 경우 사용한다.

- useEffect의 첫번째 인자는 함수이다. 해당 함수 내에서 Side Effect를 실행하면 된다.

  ```jsx
  useEffect(함수);
  ```

- useEffect의 함수가 실행되는 조건
  1. 컴포넌트 생성 후 처음 화면에 렌더링
  2. 컴포넌트에 새로운 props가 전달될 때 렌더링
  3. 컴포넌트에 상태(state)가 변경되었을 때 렌더링
- Hook을 사용할 때 주의할 점

  - 최상위에서만 Hook을 사용해야 한다.
  - React 함수 내에서만 Hook을 사용해야 한다.

- useEffect의 두번째 인자는 배열이다. 이 배열은 조건을 담고 있는데, 여기서 조건은 boolean형태의 표현식이 아닌, 어떤 값의 변경이 일어날 때를 의미한다.
  이 배열을 종속성 배열이라고 부르고, 배열 내의 종속성의 값이 변할 때, 첫번째 인자의 함수가 실행된다. 즉, 배열 내의 어떤 값이 변할 때에만, 함수가 실행된다.

  ```jsx
  useEffect(함수, 배열);
  ```

  두번째 인자에 빈배열만 넣었을 때에는, 컴포넌트가 처음 생성될 때에만 함수가 실행된다.

  ```jsx
  useEffect(함수, []);
  ```
