## React

- React는 프론트엔드 개발을 위한 Javascript 오픈소스 라이브러리 이다.

### React의 3가지 특징

1. 선언형 프로그래밍
  - 코드를 자세히 분석하지 않아도 코드를 분명히 알 수 있게 작성하는 것을 의미한다.
  - React는 한 페이지를 보여주기 위해 HTML, CSS, Javascript로 나눠서 적기보다 하나의 파일에 JSX를 활용해 명시적으로 작성하여 코드를 작성할 수 있다.

2. 컴포넌트 기반
  - 하나의 기능 구현을 위해 여러 종류의 코드를 묶어둔 컴포넌트 기반으로 개발한다.
  - 서로 독립적이고 재사용이 가능하기에, 기능 자체에 집중하여 개발할 수 있다.

3. 범용성
  - React는 Javascript 라이브러리 라고 부른다.
  - 기존에 다른 Javascript 프레임워크로 제작된 웹 어플리케이션에 React를 추가해서 작업할 수 있다.

### JSX의 규칙

- 하나의 엘리먼트 안에 모든 엘리먼트가 포함되어야 한다.

```jsx
<div>
  <div>
    <h1>hello</h1>
  </div>
  <div>
    <h2>world</h2>
  </div>
</div>
```

- 엘리먼트 class를 사용 시, class를 className으로 표기해야 한다.

```jsx
<div className="greeting">hello</div>
```

- Javascript 표현식 사용 시, 중괄호 {} 를 사용해야 한다.

```jsx
function App() {
  const name = "Taeng";
  return <div>hello, {name}</div>;
}
```

- 사용자 정의 컴포넌트는 대문자로 시작해야 한다.

```jsx
function Hello() {
    return <div>hello</div>;
}

function HelloWorld() {
    return <Hello />
```

- 조건부 렌더링에는 삼항연산자를 사용해야 한다.

```jsx
// 삼항연산자
let result = condition ? value : value2;
// condition이 true라면 value1을 false라면 value2를 반환한다

let allowAge = age > 18 ? true : false;
```

- 여러 개의 HTML 엘리먼트를 표시할 때에는 map()함수를 사용해야 한다.

```jsx
// map 함수를 사용할 때는 반드시 'key' JSX 속성을 넣어주는 것이 제일 좋다

const posts = [
  { id: 1, title: "Hello", content: "Welcome!" },
  { id: 2, title: "World", content: "Goodbye!" },
];

function Blog() {
  const content = posts.map((post) => (
    <div key={post.id}>
      <h3>{post.title}</h3>
      <p>{post.content}</p>
    </div>
  ));

  return <div>{content}</div>;
}
```

## 컴포넌트

- 컴포넌트는 하나의 기능을 구현하기 위한 여러 종류의 코드들의 묶음이다.
- 컴포넌트는 UI를 구성하는 필수 요소이다.
